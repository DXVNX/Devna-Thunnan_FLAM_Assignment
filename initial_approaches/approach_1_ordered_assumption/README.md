# Approach 1 — Ordered Assumption

## Rationale

Before deploying any optimizer, the simplest hypothesis warrants testing: that the 1500 rows of `xy_data.csv` are ordered by the parameter `t`, with row 0 corresponding to `t ≈ 6` and row 1499 to `t ≈ 60`, evenly spaced. If valid, this reduces the problem to standard `scipy.optimize.curve_fit` regression with a known `t` for each `(x, y)` pair. The check is inexpensive and worth ruling out before proceeding to more complex methods.

## Methodology

1. Constructed `t_assumed = linspace(6, 60, 1500)` and mapped it to the CSV row order.
2. Plotted `x` and `y` against `t_assumed` (see `ordering_sanity_check.png`). Under a valid ordering, both plots should exhibit smooth, continuous trends.
3. Ran `curve_fit` against `t_assumed` to quantify the fit quality.

## Results

The sanity-check plot is **jagged and noisy** — values jump erratically between consecutive rows with no discernible trend. This alone is sufficient to reject the ordering hypothesis.

The forced fit confirms the rejection quantitatively:

```
theta = 29.5823 deg
M     = -0.05000   <- pinned at the boundary of the allowed range
X     = 55.0135
RMS residual = 16.0384
```

An RMS residual of ~16 units on data spanning approximately 50–110 in `x` and 46–70 in `y` constitutes a very poor fit. The optimizer clamps `M` to the boundary of its allowed range in a failed attempt to compensate for incorrect `t` assignments. Notably, `theta` and `X` land close to the eventual true values (~30° and 55), suggesting the curve's overall shape is roughly correct but the row-to-`t` mapping is invalid.

## Key Takeaway

**The rows are shuffled.** The dataset was generated by computing 1500 `(x, y)` points from the parametric formula and then randomizing the row order. This establishes a critical constraint for all subsequent approaches:

> Each data point lies *somewhere* on the curve, but the value of `t` that produced it is unknown. Any valid approach must treat the data as an **unordered point cloud** and cannot rely on row position as a proxy for `t`.

This rules out standard regression and motivates the adoption of correspondence-free scoring methods (see Approach 2 and the Final Solution).
